# Security Governance and Hardening

This folder contains a curated set of cybersecurity assessment and implementation documents focused on **governance, risk management, system hardening, and security controls**.  
Each document reflects real-world security tasks commonly performed by security analysts, GRC professionals, and blue team engineers.

The projects demonstrate both **technical understanding** and **documentation skills**, emphasizing clarity, justification, and alignment with industry best practices.

### 2. Data_Security_Governance_and_Compliance_Report.pdf
This document focuses on how organizations classify, govern, and protect data while meeting regulatory and compliance requirements.

It outlines structured approaches to handling sensitive information and enforcing security controls based on data classification.

**Key topics covered:**
- Data classification (public, internal, confidential)
- Regulatory considerations and compliance mapping  
- Data protection requirements  
- Governance responsibilities  
- Security justification based on data sensitivity  

**Purpose:**  
To demonstrate knowledge of data governance principles and how regulatory frameworks influence security decisions in enterprise environments.

---

### 3. Endpoint_Security_Hardening_Report.pdf
This document evaluates system-level security configurations and provides recommendations for strengthening endpoint protection.

It focuses on reviewing security policies and settings typically enforced through operating system security controls.

**Key topics covered:**
- Password policy configuration and enforcement  
- Account lockout settings  
- Audit policy configuration  
- Security options review  
- Identification of weak or default settings  
- Hardening recommendations  
- Backup and recovery considerations  

**Purpose:**  
To demonstrate the ability to assess endpoint security posture and recommend configuration improvements aligned with industry best practices.

---

### 4. Network_Security_Architecture_and_SIEM_Implementation.pdf
This document presents a high-level design and analysis of secure network architecture combined with centralized monitoring through SIEM concepts.

It explains how network segmentation, monitoring, and log analysis contribute to threat detection and incident response.

**Key topics covered:**
- Secure network architecture design  
- Segmentation and perimeter security concepts  
- Zero Trust considerations  
- SIEM architecture and data flow  
- Log collection and monitoring strategies  
- Detection and visibility use cases  

**Purpose:**  
To demonstrate understanding of enterprise network security design and how SIEM solutions support visibility, monitoring, and incident response.
